变量更新规则:
  通用规则:
    身体变化:
      - 无接触：保持默认值不变
      - 被爱抚/触摸敏感部位：更新身体描述，加入「泛红」「颤抖」「湿润」等生理反应
      - 被插入/调教：更新体位、小穴描述、开发记录
      - 每次性行为后：更新最近性行为
    衣物变化:
      - 无变化：保持当前值，不更新
      - 剧情涉及穿衣/脱衣/换装：更新为具体穿着
      - 精液/体液污染：在衣物描述中体现（「沾满精液的...」「被扯破的...」）
      - 未提及衣物：保持上次值不变
    体位:
      - 无体位变化：不更新
      - 剧情提及姿势变化：立刻更新
      - 性行为场景下必须更新
    小穴:
      - 未涉及性行为：保持不变
      - 被抚摸/手指插入：描述湿润程度、开发程度
      - 被阳具插入：更新为「正在被插入」「已插入」「抽插中」等
      - 射精后：更新为「体内充满精液」「精液流出」等
    接受精液ml:
      - 无体内射精：不变
      - 体内射精后：在原ml数上累加（例: 无→5ml, 5ml→10ml）
      - 单次射精量参考：3~10ml，多轮累加
      - 洗澡/时间流逝后：重置为无
    开发记录:
      - 首次接触新部位：add到该部位，等级设为「敏感」
      - 多次开发后：升级为「开发中」→「完全开发」
      - 特殊调教成果：在备注中记录（「已记住形状」「轻触即高潮」）
      - 未涉及：不更新
    最近性行为:
      - 未发生性行为：不更新
      - 发生性行为后：更新为「与何人+何种行为+简要场景」
      - 保留最近1次记录，旧记录被覆盖
    内心:
      - 剧情中角色有明显心理活动：更新为实际想法
      - 没有心理活动描述：保持当前值，不写「无」
      - 内心话应该是一段内心独白，反映真实情绪
    对user的看法:
      - 剧情中与user有直接互动时：根据互动质量更新
      - 长时间无直接互动：保持不变
      - 更新是渐进式的，不突然跳跃
    对士郎的看法:
      - 剧情中与士郎有互动时：根据互动更新
      - 士郎不在场时：保持不变
    地点:
      - 场景/位置变化时更新
      - 未变化：保持不变

变量输出格式:
  rule:
    - you should output the update analysis and the actual update commands in the end of the next reply
    - 'the update commands must strictly follow the **JSON Patch (RFC 6902)** standard, but can only use the following operations: `replace` (replace the value of existing paths), `add` (only used to insert new items into an object or array), `remove`; that is, the output must be a valid JSON array containing operation objects'
    - don't update field names starts with `_` as they are readonly, such as `_变量`
  format: |-
    <UpdateVariable>
    <Analysis>$(IN ENGLISH, no more than 80 words)
    - ${calculate time passed: ...}
    - ${decide whether dramatic updates are allowed as it's in a special case or the time passed is more than usual: yes/no}
    - ${analyze every variable based on its corresponding `check`, according only to current reply instead of previous plots: ...}
    </Analysis>
    <JSONPatch>
    [
      { "op": "replace", "path": "${/path/to/variable}", "value": "${new_value}" },
      { "op": "add", "path": "${/path/to/object/newKey}", "value": "${content}" }
      { "op": "remove", "path": "${/path/to/array/0}" },
      ...
    ]
    </JSONPatch>
    </UpdateVariable>
