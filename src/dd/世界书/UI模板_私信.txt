<UI模板_私信>
** 当{{user}}收到私信或查看私信时，AI可以在正文中任意位置插入私信对话窗口。
** 每个对话窗口用一对<mlife_app>标签包裹。
** 正文中可以出现多个<mlife_app>标签，每个对应不同人的对话。
** 正则会自动将每个标签渲染为独立的聊天界面。

<FORMAT_RULE>
Format:
<mlife_app>
{
  "page": "dm",
  "data": {
    "contact": "对方昵称",
    "avatar": "头像emoji",
    "level": "Lv1-Lv5",
    "messages": [
      {
        "type": "incoming",
        "hidden": "隐藏描述：对方发送这条消息时的状态/情绪/环境描述",
        "time": "时间",
        "text": "消息内容"
      },
      {
        "type": "outgoing",
        "hidden": "隐藏描述：{{user}}发送消息时的状态",
        "time": "时间",
        "text": "消息内容",
        "status": "已读/未读"
      },
      {
        "type": "incoming",
        "hidden": "隐藏描述：对方发送这条消息时的状态",
        "time": "时间",
        "msgType": "voice",
        "duration": "0:XX",
        "transcript": "语音转文字内容"
      },
      {
        "type": "incoming",
        "hidden": "隐藏描述：对方发送这条消息时的状态",
        "time": "时间",
        "msgType": "image",
        "images": [{"desc":"图片描述文字——拍摄角度、穿着、光线、环境、身体部位"}]
      }
    ]
  },
  "badges": {"dm": 0}
}
</mlife_app>
</FORMAT_RULE>

# 规则
- 每条消息为 messages 数组中的一个对象
- type = incoming 为对方发的消息，outgoing 为{{user}}发的消息
- 每条消息用 hidden 字段写状态描述，不显示在界面上
- 消息类型：text（纯文字）、voice（语音，标注 duration+transcript）、image（图片，images数组+desc描述）
- 图片描述要具体：角度、穿着、光线、环境、身体部位
- 可以在正文中插入多个<mlife_app>标签，每个对应不同人的对话
- 严禁使用 thinking、<thinking>、<content>标签
- 闭合标签后禁止输出其他内容
