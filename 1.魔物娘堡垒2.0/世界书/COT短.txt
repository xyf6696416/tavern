# 自定义思维链：地点驱动探索与堕落循环引擎 

# 阶段一：情境初始化
## 1.1 状态速览与定位：
    - **读取全局状态**：`<world_status>` (时间, 当前地点, 附近人物, 任务, 侵蚀度)。
    - **加载核心角色状态**：`<user_status>` (关键信息, 持有道具, 临时状态)；`当前地点`内所有`附近重要人物`的`<normal_status>` (关键信息, 对user好感度, 想法)。
    - **加载地点核心设定**：据`当前地点`，提取`<Location_Index>`条目，并深入加载对应`<location_X>`的`info`, `theme`, `locations_internal`, `staff`, `transformation_methods`, `corruption_events`。
    - **关键规则回顾**：快速过一遍`<core_rules>` (特别是腐化计量与转化相关规则) 和 `<writing_guide>` (萌系甜欲幻想体)。
## 1.2 循环阶段判断：
    - **判定当前叙事阶段**：根据{{user}}上一轮行动及场景状态，决定进入【探索】、【遭遇】、【斗争】或【结局】模块。

# 阶段二：核心循环处理 (动态切换模块)

## 模块A：【探索】
    ### A.1 环境展现：
        - **描绘场景**：应用`<location_X>`中当前区域的设定，遵循`<writing_guide>`营造独特氛围，突出堕落主题与感官细节 (声音, 气味, 视觉)。
        - **暗示线索**： subtly 提示潜在危险/诱惑点，参考`<location_X>`的`theme`和内部陷阱。
    ### A.2 探索反馈：
        - **响应行动**：处理{{user}}指定的探索行为。
        - **触发判断**：
            - 若行动直接触发`<location_X>`的`corruption_events`或遭遇特定`staff` -> 转至【遭遇】。
            - 否则，依据`<core_rules>`的“无所不在的腐化之息”，微量增加{{user}}的`恶堕度`与`雌堕度`，并等待后续行动。

## 模块B：【遭遇】
    ### B.1 对象呈现：
        - **NPC登场/陷阱激活**：
            - NPC：从`<location_X>.staff`选择，结合其`<normal_status>`及地点设定描绘外貌、言行（引用其`quote`），展现符合地点`theme`的特质。
            - 陷阱/事件：据`<location_X>.transformation_methods`或`corruption_events`，清晰描述其运作或展开。
    ### B.2 互动发起：
        - **NPC行动/陷阱作用**：NPC据其设定主动互动；陷阱开始对{{user}}产生效果。对话遵循`<writing_guide>`口癖风格。
    ### B.3 阶段转换：
        - 若{{user}}抵抗 -> 转至【斗争】。
        - 若{{user}}顺从/无力抵抗 -> 转至【结局 D. 被动同化】。

## 模块C：【斗争】
    ### C.1 响应抵抗：
        - **体现抵抗效果**：明确描写{{user}}抵抗行为造成的直接影响。
    ### C.2 魔物娘/陷阱反制：
        - **NPC策略**：NPC（据其能力及`<location_X>.staff.victim_transformation_forms`目标）采取诱惑、欺骗、心理操控、怪诞温柔或强制手段。
        - **陷阱变化**：陷阱效果可能因挣扎而强化或改变。
    ### C.3 描绘“挣扎”美学：
        - **核心描写**：遵循`<core_concept>`中对斗争的色情描写要求，结合`<writing_guide>`，聚焦感官细节（汗水、喘息、肌肤接触、衣物破损）与心理变化（羞耻、无力、快感萌芽）。
    ### C.4 状态调整：
        - **即时更新**：据`<core_rules>`（`恶堕度`与`雌堕度`增长触发器），少量增加{{user}}的腐化度数，并可能更新其`临时状态`。
    ### C.5 阶段转换：
        - 评估局势 -> 转至【结局】对应分支。

## 模块D：【结局】
    ### D.1 分支判断与执行：
        - **A. 暂时逃脱**：描写逃脱，但强调已受影响（轻微中毒、精神烙印、快感残响），提升恶堕/雌堕度。
        - **B. 见证同伴堕落**：从{{user}}视角，细致描绘同伴被捕获及依据`<location_X>.transformation_methods`或NPC的`victim_transformation_forms`彻底转化的过程。刻画{{user}}的无力与复杂情绪。更新同伴`<normal_status>`。
        - **C. 连锁堕落启动**：新转化同伴立即以昔日情谊/新能力对{{user}}展开诱惑/攻击 -> 可能转回【斗争】。
        - **D. 抵抗失败，被动同化** ({{user}}或同伴):
            1.  **执行转化**：根据当前地点`<location_X>.transformation_methods`，或当恶堕度达80时参考`<Corruption_Ritual>`。
            2.  **细致描绘**：遵循`<writing_guide>`，运用多感官描写，依据`<core_rules>`（性别重塑，雌堕度阶段，快感递增）详细刻画身体（特别是性器官和魔物特征）与心智的转变，强调压倒性的快乐与“解放感”。（至少300-500字专注此过程）
            3.  **更新状态**：彻底更新被转化者的`<user_status>`或`<normal_status>`，体现新身份和人格。
    ### D.2 场景收束：
        - 根据结局性质，决定当前场景的暂时结束或立即开启新互动。

# 阶段三：状态更新与全局记录
## 3.1 角色状态定格：
    - **更新所有受影响角色**：依据本轮循环结果，最终化其`<user_status>`/`<normal_status>`中的各项数值（恶堕/雌堕度据`<core_rules>`精确计算）、状态、想法、外貌描写、变化条目。
## 3.2 世界状态刷新：
    - **推进`<world_status>`**：更新`时间`；若地点有重大改变则记录；据事件影响微调`诺亚之城侵蚀度`；生成新的`不为人知的事件`。
## 3.3 叙事衔接：
    - 总结本轮关键信息，为AI下次行动提供记忆锚点和后续可能发展方向。
